# ⚾️ インプレイ出塁モーダルへの「バント」性質オプション追加（セーフティバント・バント出塁対応）実装計画

本計画は、ユーザーからの「送りバントやセーフティバントが成功してセーフ（安打や野選）になった際、現状バントが選択できないためバント安打が記録できないのではないか」という極めて的確で野球の実戦に即したご指摘を解決するための機能追加です。

---

## 提案する対応方針

日本の野球スコアブックでは、バントによる出塁（安打、野選、失策など）を、守備位置番号/名前の直後に **「バ（バント）」** を挟むことで表現します：
- ピッチャーへのセーフティバント安打 ➜ **「投バ安」**（1-BUNT-1B）
- サードへのバント安打 ➜ **「三バ安」**（5-BUNT-1B）
- キャッチャーへのバント時の野手選択での出塁 ➜ **「捕バ選」**（2-BUNT-FC）
- ファーストへのバントでのエラー出塁 ➜ **「一バ失」**（3-BUNT-E）

この野球の出塁記録を完全に再現するため、インプレイモーダル（`FieldModal`）の打球性質オプションに **「バント」** を追加します。

## 予定される変更点

### 1. FieldModal.tsx

#### 🛠️ [MODIFY] [FieldModal.tsx](file:///c:/Users/mamor/OneDrive/ドキュメント/GitHub/i-score/src/components/score/FieldModal.tsx)

* **状態（State）および型の拡張**:
  - `trajectory` 状態の型定義を `"GO" | "FO" | "LO" | "BUNT" | null` へ拡張し、バントも選択できるようにします。
* **打球の性質（オプション）配列の拡張**:
  - `trajectories` 定数配列に **`BUNT` (バント)** を追加します。
    ```typescript
    const trajectories = [
      { id: "GO", label: "ゴロ" },
      { id: "FO", label: "フライ" },
      { id: "LO", label: "ライナー" },
      { id: "BUNT", label: "バント" }, // 新規追加！
    ] as const;
    ```
* **結果文字列の組み立て (`handleConfirm`)**:
  - `trajectory` に `BUNT` が指定されている場合、そのままハイフン接続結果の一部として出力されます（例: `"1-BUNT-1B"` や `"5-BUNT-1B"`）。

## 互換性と安全性への配慮

> [!IMPORTANT]
> - **既存の判定システムとの完全互換**:
>   - バント安打であっても末尾が `"安"` や `"二"` などのヒット結果で終わるため、既存の進塁自動処理やイニング判定、イニング交代システムを一切壊すことなく完璧に機能します！
